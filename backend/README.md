# Pakalorie Backend

FastAPI service for the P1 Final backend lane:

- Food database API backed by PostgreSQL 16 with `pg_trgm` and `vector` enabled.
- Server-side Gemini image recognition, so the API key stays out of the mobile bundle.
- Grounded calorie engine that retrieves real nutrition rows before generating a breakdown.

## Local Setup

```powershell
cd backend
uv sync
Copy-Item .env.example .env
docker compose up db -d
uv run alembic upgrade head
uv run python -m scripts.seed_foods
uv run uvicorn app.main:app --reload
```

Health check:

```powershell
curl http://localhost:8000/healthz
```

Expected response:

```json
{"status":"ok"}
```

## Seed Data

`data/desi_seed.json` is the 30-dish curated Pakistani core copied into the repo for
reproducibility. `data/usda_foundation_sample.json` is a small filtered extract from
USDA FoodData Central Foundation Foods 04/2026, generated by:

```powershell
uv run python -m scripts.build_usda_extract --source-dir <extracted-fdc-csv-folder>
```

The seed command imports both sources and produces exactly 160 `foods` rows
(30 `desi_v1` + 130 `usda`). See [`docs/DATA_NOTES.md`](docs/DATA_NOTES.md) for how
to read each source.

## Tests

Two modes:

```powershell
# Unit tests only (no database needed) — fast, always runnable
uv run pytest -m "not integration"

# Integration tests against a live, seeded database
# 1. open the SSH tunnel so localhost:5432 -> VPS Postgres
# 2. ensure backend/.env DATABASE_URL points at it
uv run pytest -m integration
```

A plain `uv run pytest` runs both; the integration suite **auto-skips** when no
database is reachable at `DATABASE_URL`, so it is safe in CI without a database.
